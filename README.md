# 📌 Project Title

Manual Testing Documentation – TutorialNinja E-Commerce Website

## 📖 Project Overview

- This project contains complete manual testing documentation for the demo e-commerce website:
tutorialninja.com

- The objective of this project is to validate core functionalities of the application such as user authentication, product search, cart operations, and checkout process through structured testing techniques.

- The project demonstrates practical knowledge of:

1. Test planning
2. Test case design
3. Requirement traceability

## 🎯 Scope of Testing

The following modules were tested:

- Home Page
- User Registration
- Login / Logout
- Search Functionality
- Product Detail Page
- Add to Cart
- Wishlist
- Checkout Flow
- Contact Us Form
- Forgot Password

## 🛠️ Tools Used

- Test Case Management: Excel / Google Sheets
- Browser Testing: Google Chrome
- Operating System: Windows 10

## 📊 Testing Artifacts Included
- ✅ Test Plan
Scope, objectives, and strategy
Entry & exit criteria
Testing types and environment
- ✅ Test Cases (40+)
Covers all modules
Includes positive & negative scenarios
Contains priority and severity
- ✅ RTM (Requirement Traceability Matrix)
Maps all requirements to test cases
Ensures 100% coverage


## 🚀 Key Highlights
- End-to-end coverage of major e-commerce workflows
- Focus on edge cases and validations

## 📌 How to Use
- Review the Test Plan to understand scope and strategy
- Execute Test Cases from the test case document
- Use RTM to track requirement coverage
